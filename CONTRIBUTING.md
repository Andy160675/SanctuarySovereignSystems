# Contributing to SanctuarySovereignSystems

Thank you for your interest in contributing to SanctuarySovereignSystems!

## Code of Conduct

We are committed to providing a welcoming and inclusive environment for all contributors.

## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker to report bugs or suggest features
- Provide clear descriptions and reproduction steps for bugs
- Check existing issues to avoid duplicates

### Pull Requests

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes following our coding standards
4. Include tests for new functionality
5. Update documentation as needed
6. Submit a pull request with a clear description

### Coding Standards

- Follow existing code style and conventions
- Use meaningful variable and function names
- Add comments for complex logic
- Ensure all tests pass before submitting

### Governance Requirements

For changes affecting governance or security:

1. Update relevant templates in `docs/templates/`
2. Document in the traceability matrix
3. Update threat models if security-related
4. Add evidence artifacts to `evidence/` directory

### Testing

- All code changes must include appropriate tests
- Run the full test suite before submitting
- Ensure linting passes

### Documentation

- Update README.md if adding features
- Document new templates or processes
- Include examples where helpful

## Review Process

All contributions go through:

1. Automated CI checks (linting, tests)
2. Code owner review (see CODEOWNERS)
3. Security review for sensitive changes
4. Phase gate approval for major changes

## Questions?

Open an issue for questions or clarifications.
