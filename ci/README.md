# CI/CD Configuration

This directory contains CI/CD pipeline configurations.

## Purpose

Automated pipelines for:
- Linting and code quality checks
- Running test suites
- Security scanning
- Build automation
- Deployment

## Guidelines

- All code changes must pass CI checks
- Security scans must be clean
- Test coverage requirements enforced
- Failed builds block merges
- Document pipeline stages

## GitHub Actions

GitHub Actions workflows are located in `.github/workflows/`
