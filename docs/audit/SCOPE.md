# Audit Scope: Sovereign Doctrine & History

## Purpose
To establish a verifiable provenance for the Sovereign System's core logic and its evolution over time.

## Included Artefacts
- **CANON/**: Core protocols and deployment logic.
- **governance/**: The Diamond Doctrine and authority schemas.
- **CONSTITUTION.md**: Foundational rules.
- **SOP_DAILY.md**: Operational rhythm.
- **evidence/**: Captured telemetry and validation reports.

## Exclusions
- Third-party library source code (e.g., `node_modules`).
- Transient log files not part of the permanent ledger.
- Individual user download folders.
